# Симуляция

Консольное приложение симулирует жизнь мира, в котором живут козы и лисы. Они живут на карте, среди объектов которой трава, камни, деревья, а также мир огорожен стеной. 

У каждого из двух видов животных есть своя задача: у лис - съесть всех коз, у коз - съесть всю траву (на месте съеденной травы появляются  камни). Тот, кто справился со своей задачей, объявляется победителем. 

У животных есть энергия на ход, которую можно  потратить как на передвижение, так и на поедание цели. Съедание травы немного пополняет здоровье коз. В среднем козы побеждают чаще, но бывает, что побеждают и лисы. Многое зависит от того, сколько представителей каждого лагеря появится при инициализации карты: этот показатель относительно случаен. 

## Внутреннее устройство

Проект состоит из следующих модулей:
- map. Включает в себя классы карты и клеток 
- renderer. Отвечает за отрисовку карты после каждого хода 
- entities. Включает классы животных (creatures) и статических объектов. В классе creatures реализован метод совершения хода животным 
- actions. Включает в себя функционал для инициализации карты, совершения хода всеми существами. В файле BFS реализован алгоритм поиска животным пути до ближайшей цели (алгоритм поиска в ширину)
- simulation. Отвечает за игровое меню (взаимодействие с пользовательским вводом)


## Старт симуляции

Чтобы начать взаимодействие с приложением следует запустить файл ```main.py``` в катологе проекта. 
При старте симуляции будет инициализировано игровое поле размером 20 на 10 ячеек, параметры скорости и жизни существ изначально заданы. В случае необходимости их можно изменить (в конструкторе объекта карты, в инициализаторе)

Возможные действия:
- `s` - запустить один ход симуляции: каждое из существ на карте сделает свой ход
- `q` - запустить цикл ходов. После каждого из них будет отрисовано текущее положение на карте
- `p` — поставить цикл ходов на паузу (возобновить) - `m` - выход из меню паузы в главное меню
- `e` — завершить работу приложения.

Приложение завершает свою работу либо по требованию пользователя, либо после выявления победителя симуляции.
